# VK-бот «Точка на карте»

Бот для сообщества VK: проводит пользователя по анкете, подбирает страны по матрице `countries.json` и отправляет заявку администратору в VK.

## Что было не готово к серверу

- Не было Dockerfile и `docker-compose.yml`, поэтому проект нельзя было нормально поднять как сервис.
- Не было `.gitignore` и `.dockerignore`; был риск случайно отправить `.env` с токеном в git или в Docker build context.
- Бот использовал `random_id=0` для всех сообщений VK. Для продакшена нужен уникальный `random_id`.
- Ошибки печатались через `print`, без нормальных логов для `docker logs`.
- При падении longpoll или сетевой ошибке процесс мог завершиться без восстановления.
- Валидация входных данных была минимальной.
- Данные стран нужно сверить с реальной коммерческой таблицей: в текущем `countries.json` у части стран пустые поля, из-за чего они не смогут дать совпадение 4/4.

## Секреты

Токены и пароли не хранятся в репозитории. На сервере создайте `.env` из шаблона:

```bash
cp .env.example .env
nano .env
```

Обязательные переменные:

- `VK_TOKEN` — токен сообщества VK с правами на сообщения и управление сообществом.
- `ADMIN_VK_ID` — числовой VK ID администратора, которому отправляются заявки.

Опциональные переменные:

- `VK_GROUP_ID` — ID группы VK. Можно оставить пустым, бот получит ID по токену.
- `LOG_LEVEL` — уровень логов, например `INFO` или `DEBUG`.

Да, токен лучше вставлять уже на сервере в `.env`, а не хранить в коде и не отправлять в GitHub.

## Запуск через Docker Compose

```bash
docker compose up -d --build
docker compose logs -f vk-bot
```

Остановка:

```bash
docker compose down
```

Обновление после новых коммитов:

```bash
git pull
docker compose up -d --build
```

## Локальный запуск без Docker

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python bot.py
```

## Настройка VK

1. Включить сообщения сообщества.
2. Создать ключ доступа сообщества с правами на сообщения и управление сообществом.
3. Вставить ключ в `VK_TOKEN`.
4. Написать боту в личные сообщения сообщества: `Начать подбор`.

Если VK не дает ключу право управления, бот автоматически использует резервный
опрос непрочитанных сообщений. Для штатного Long Poll право управления все равно
следует добавить.

## Логика подбора

- Пользователь выбирает до 2 вариантов в каждом блоке.
- Внутри блока работает логика ИЛИ.
- Между блоками работает логика И.
- Если пользователь выбрал `ПМЖ`, также учитывается `Золотая виза`.
- `План Б` не исключает страну.
- Страны ранжируются по совпадениям; клиент всегда получает до трёх лучших вариантов.
- При равном количестве совпадений выше учитываются цель, бюджет, финансирование и мотивация.
- Высокий бюджет включает более доступные программы, а для ВНЖ учитываются близкие программы ПМЖ и золотой визы.
- Результат оформляется отдельным списком с флагами стран и одинаково отображается клиенту и администратору.
- Имя, телефон, ответы и ссылка на профиль отправляются администратору в VK.

## Что ещё нужно проверить перед коммерческим запуском

- Сверить `countries.json` с исходной таблицей услуг, бюджетов и стран.
- Пройти полный сценарий в VK с тестовой группой.
- Решить, нужна ли CRM/таблица для заявок. Сейчас заявки уходят администратору в VK.
- Если бот будет запускаться в нескольких контейнерах, вынести `SESSIONS` из памяти в Redis или базу данных.
